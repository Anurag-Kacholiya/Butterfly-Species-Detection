\begin{abstract}
Identifying butterfly species from photographs is a classic \textit{Fine-Grained Visual Categorization (FGVC)} problem, where many species exhibit highly similar visual characteristics due to biological mimicry. The task is further complicated by significant intra-species variations such as differences in sex, wing wear, lighting conditions, and pose.

In this project, we develop a full-stack Streamlit-based web application capable of classifying 100 butterfly species while simultaneously providing additional interpretability and ecological information, including IUCN conservation status, Wikipedia summaries, and Grad-CAM visual explanations during inference.

To evaluate model performance systematically, we conduct an extensive ablation study using three transfer-learning strategies on EfficientNet-B0~\cite{tan2019efficientnet}:
\begin{itemize}
    \item[(A)] Linear Probing
    \item[(B)] Last-Block Fine-Tuning
    \item[(C)] Full Fine-Tuning
\end{itemize}

We also compare these approaches against a zero-shot CLIP baseline~\cite{radford2021learning}.

Experimental results show that the zero-shot CLIP model achieves only 30.8\% Top-1 accuracy, highlighting the difficulty of the FGVC task. Strategy A (Linear Probing) attains 92.20\% Top-1 accuracy while training only 1,300 parameters. Strategy B (Last-Block Fine-Tuning), using 2.9 million trainable parameters, achieves the best trade-off between efficiency and performance with \textbf{97.0\%} Top-1 accuracy, 99.2\% Top-3 accuracy, and 96.92\% Macro-F1 score. Notably, this matches the performance of the fully fine-tuned model (Strategy C: 5.3 million parameters, 97.0\% Top-1 accuracy) while requiring nearly half the trainable parameters.

Furthermore, Grad-CAM visualizations demonstrate that the model primarily focuses on biologically meaningful wing venation and pattern structures rather than background habitat information. A UMAP-based feature space analysis also reveals compact and well-separated clusters for individual butterfly species, indicating strong discriminative representation learning.

\end{abstract}

\section{The Fine-Grained Classification Challenge}

Butterfly species identification is far harder than standard image classification. The dataset used in this work~\cite{gpiosenka} comprises \textbf{12,594 training}, \textbf{500 validation}, and \textbf{500 test} images spanning exactly \textbf{100 species}, each with 224$\times$224 pixel images. Key challenges include:

\begin{itemize}
    \item \textbf{Biological mimicry:} Species such as Monarch and Viceroy have co-evolved nearly identical wing colouration to deceive predators. The classifier must resolve these pairs using only subtle vein spacing or spot size.
    \item \textbf{Sexual dimorphism:} Male and female individuals of the same species often display radically different colouration (e.g., Adonis Blue).
    \item \textbf{Highly variable imaging conditions:} Field photographs vary in lighting, occlusion, wing folding, and camera angle.
    \item \textbf{Class imbalance:} Common species have more training examples than rare ones, skewing naive classifiers.
\end{itemize}

The dataset is publicly available on Kaggle~\cite{gpiosenka}. The training pipeline runs on a single NVIDIA Tesla T4 GPU (15.6~GB VRAM) via Kaggle Notebooks.

\section{Approach 1: Zero-Shot CLIP}

OpenAI's CLIP~\cite{radford2021learning} is a dual-encoder model that jointly embeds images and text. Zero-shot inference requires no additional training: we compute the cosine similarity between the image embedding and 100 text embeddings of the form \emph{``A photo of a \{species\}, a type of butterfly.''} The highest-scoring text label is taken as the prediction.

\subsection{Ensemble Prompting}

To give CLIP every advantage, we constructed an ensemble of seven prompt templates per species, including:
\begin{enumerate}
    \item \textit{``A photo of a \{species\}, a type of butterfly.''}
    \item \textit{``A close-up field photograph of a \{species\} butterfly.''}
    \item \textit{``The \{species\} butterfly on a flower.''}
\end{enumerate}

The per-species embedding is the mean of all seven text embeddings before normalisation.

\subsection{Why Zero-Shot Failed}

Despite ensemble prompting, CLIP achieved only \textbf{30.8\% Top-1} and \textbf{44.2\% Top-3} accuracy. UMAP visualisation of CLIP image embeddings reveals heavily overlapping clusters — the model cannot discriminate the 100 species at the embedding level. CLIP's visual vocabulary is oriented towards common objects and web-scale concepts, not the micro-structural details (ocelli placement, submarginal bands, forewing apex shape) that distinguish butterfly species. This motivates learning a specialist representation via fine-tuning.

\section{Approach 2: Transfer Learning on EfficientNet-B0}

EfficientNet-B0~\cite{tan2019efficientnet} is a compound-scaled convolutional network with 4.14~M parameters (as loaded via \texttt{timm}~\cite{rw2019timm} with ImageNet-1k pre-training). Its architecture terminates with a Global Average Pooling layer that yields a 1,280-dimensional feature vector before the linear classification head.

\subsection{Data Augmentation}

To maximise data efficiency, training images pass through an aggressive augmentation pipeline:
\begin{itemize}
    \item \texttt{RandomResizedCrop(224, scale=(0.7, 1.0))}
    \item \texttt{RandomHorizontalFlip}, \texttt{RandomVerticalFlip(p=0.1)}
    \item \texttt{ColorJitter(brightness=0.3, contrast=0.3, saturation=0.2, hue=0.05)}
    \item \texttt{RandomRotation(15°)}
    \item ImageNet mean/std normalisation
\end{itemize}

Validation and test images use only \texttt{Resize(256)}, \texttt{CenterCrop(224)}, and normalisation. Training is performed with \texttt{batch\_size=32}, \texttt{num\_workers=2}, and \texttt{pin\_memory=True} on a T4 GPU.

\subsection{Phase 1 — Linear Probe (Strategy A)}

All backbone weights are frozen. Only the final 1,280→100 linear layer is trained.

\begin{itemize}
    \item \textbf{Trainable parameters:} $\approx$1,300 (out of 4.14~M total)
    \item \textbf{Optimiser:} Adam, LR = $10^{-3}$
    \item \textbf{Epochs:} 15 (early stopping on validation accuracy)
    \item \textbf{Best validation accuracy:} \textbf{92.20\%} (epoch 12)
    \item \textbf{Test Top-1:} \textbf{92.20\%}
\end{itemize}

The rapid convergence — epoch 1 already achieves 82.8\% validation accuracy — confirms that ImageNet-pretrained EfficientNet-B0 features transfer immediately to fine-grained insect classification. The frozen backbone acts as a strong regulariser, producing no divergence between training (93.5\% at epoch 15) and validation accuracy.

\textbf{Limitation:} Linear probing plateaus below 93\%. The frozen backbone cannot adapt its feature detectors to the micro-structural cues that distinguish mimicry pairs, motivating Phase 2.

\subsection{Phase 2 — Fine-Tune Last Block (Strategy B)}

Starting from the Phase 1 checkpoint, we unfreeze \texttt{blocks[6]} (the final MBConv block), \texttt{conv\_head}, \texttt{bn2}, and the classifier head:

\begin{itemize}
    \item \textbf{Trainable parameters:} $\approx$2.9~M (70\% of total)
    \item \textbf{Optimiser:} Adam, LR = $10^{-4}$ (reduced to avoid catastrophic forgetting)
    \item \textbf{Epochs:} 15 additional
    \item \textbf{Best validation accuracy:} \textbf{96.20\%} (epoch 12)
\end{itemize}

\begin{figure*}[t]
    \centering
    \includegraphics[width=\linewidth]{effnet_training_curves.png}
    \caption{Training curves for Phase 1 (linear probe, left) and Phase 2 (last-block fine-tune, right). Phase 1 converges rapidly in $\sim$5 epochs. Phase 2 continues improving validation accuracy from 93\% to a peak of 96.2\%, with no evidence of overfitting.}
    \label{fig:training_curves}
\end{figure*}

\textbf{Why unlocking only the last block works:} Early EfficientNet blocks (blocks[0]–[5]) encode generic low-level features — Gabor-like edge detectors, colour blobs, textures — that transfer well across all vision domains. Block[6] encodes the highest-level semantic representations before pooling. Unlocking it lets the model remap its feature vocabulary to butterfly-specific cues (wing venation, eyespots, colour bands) without disturbing the foundational visual hierarchy. Figure~\ref{fig:training_curves} shows the training dynamics for both phases.

\subsection{Strategy C — Full Fine-Tuning}

As an ablation upper bound, we also fine-tune all 5.3~M parameters with the same LR = $10^{-4}$ schedule for 15 epochs:

\begin{itemize}
    \item \textbf{Best validation accuracy:} \textbf{97.40\%}
    \item \textbf{Test Top-1:} \textbf{97.00\%}
    \item \textbf{Macro F1:} \textbf{96.87\%}
\end{itemize}

Strategy C matches Strategy B in test accuracy but requires nearly double the trainable parameters and trains more slowly due to gradient computation across all layers.

Table~\ref{tab:ablation} summarises all four strategies evaluated on the held-out 500-image test set. We also include the dataset author's Keras/TensorFlow EfficientNet-B0 model~\cite{gpiosenka} as a reference ceiling (zero additional training on our part).

\begin{figure}[H]
    \centering
    \includegraphics[width=\linewidth]{effnet_ablation_chart.png}
    \caption{Ablation bar chart comparing test Top-1 accuracy against trainable parameter count. Strategies B and C reach the same accuracy; Strategy B is more parameter-efficient.}
    \label{fig:ablation_bar}

\textbf{Key findings:}
\begin{enumerate}
    \item \textbf{Strategy B = Strategy C in accuracy, at lower cost.} Both reach 97.0\% Top-1 on the test set. Fully fine-tuning 5.3~M parameters yields no benefit over fine-tuning 2.9~M, confirming that the early layers are already optimal for this domain.
    \item \textbf{The biggest gain is A → B (+4.8\%).} The jump from linear probing to last-block fine-tuning is far larger than B → C (0\%), proving that task-specific adaptation in the final semantic layers is the critical bottleneck.
    \item \textbf{Top-3 accuracy (99.2\%) is near-flawless.} The correct species is in the model's top-3 candidates for 99.2\% of test images, making the classifier highly suitable for field-identification apps where experts can confirm the top prediction.
    \item \textbf{The 0.6\% gap vs the Keras baseline} (97.60\% vs 97.00\%) is a deliberate engineering trade-off: the author's model used full end-to-end training on dedicated infrastructure. Our model is a compact 16~MB PyTorch file optimised for live Streamlit inference.
\end{enumerate}

\section{Interpretability Analysis}

\begin{figure}[H]
    \centering
    \includegraphics[width=\linewidth]{gradcam_visualisation.png}
    \caption{Grad-CAM for 4 correct (top) and 4 incorrect (bottom) predictions. Activation is consistently concentrated on wing surfaces, margins, and thorax — the biologically relevant discriminating regions.}
    \label{fig:gradcam}
\end{figure}

\subsection{Grad-CAM Heatmaps}

Grad-CAM~\cite{selvaraju2017gradcam} back-propagates the gradient of the predicted class score through the last convolutional layer (\texttt{blocks[-1]}) to produce a spatial activation map highlighting the regions most influential to the prediction.

Figure~\ref{fig:gradcam} shows four correct and four incorrect predictions. In all cases, activation focuses on \textbf{wing patterns, submarginal bands, ocelli, and the thorax} — the exact biological features used by human taxonomists. The background (leaves, stems, soil) receives near-zero activation, confirming the model has not learned habitat spurious correlations.

Even in failure cases, the model looks at the right anatomical region. Errors are biologically plausible (mimicry pairs), not random. This ``fail gracefully'' property is critical for a field-identification tool.

\subsection{Layer-wise Grad-CAM}

To demonstrate how attention evolves across network depth, we extract Grad-CAM heatmaps at three representative blocks and visualise them for the same input images (Figure~\ref{fig:gradcam_layerwise}):

\begin{table}[H]
\centering
\renewcommand{\arraystretch}{1.1}
\begin{tabular}{clp{3.8cm}}
\toprule
\textbf{Block} & \textbf{Depth} & \textbf{Role} \\
\midrule
\texttt{blocks[1]} & Early  & Low-level edges, textures, colour blobs \\
\texttt{blocks[4]} & Middle & Shape parts, wing segments \\
\texttt{blocks[6]} & Late (fine-tuned) & Species-specific discriminative patterns \\
\bottomrule
\end{tabular}
\end{table}


\textbf{blocks[1] (Early):} The heatmap spreads broadly across the entire image — edges, background foliage, and butterfly body all activate simultaneously. The network is performing low-level feature detection analogous to Gabor-like filters. At this stage there is no species-specific reasoning; the model simply responds to oriented edges and colour boundaries wherever they appear.

\textbf{blocks[4] (Middle):} Attention begins to narrow towards the butterfly body and wings. Background suppression becomes apparent. The model is now detecting mid-level features: the overall wing outline, broad colour regions, and approximate body shape. The spatial attention is more selective than in the early block but is not yet focused on the fine-grained cues that distinguish species.

\begin{figure}[H]
    \centering
    \includegraphics[width=\linewidth]{gradcam_layerwise.png}
    \caption{Layer-wise Grad-CAM at blocks[1] (early), blocks[4] (middle), and blocks[6] (late/fine-tuned) for the same input images. Heatmaps progress from broad whole-image activations to tight, species-specific wing-feature highlights.}
    \label{fig:gradcam_layerwise}
\end{figure}

\textbf{blocks[6] (Late --- fine-tuned):} Activation is tightly concentrated on fine-grained discriminative features --- specific wing bands, colour patches, eyespots, and margin patterns. This is precisely the block we unlocked during Phase 2 fine-tuning; its representations have been adapted from ImageNet semantics to butterfly-specific diagnostic anatomy.

\textbf{Why this matters:} The layer-wise progression provides strong empirical evidence that fine-tuning only \texttt{blocks[6]} is sufficient. Blocks 0--5 already encode a rich feature pyramid that transfers well to the butterfly domain. By unlocking only the final block, we ``sharpen the lens'' for species-level discrimination without destabilising the foundational visual vocabulary learned from ImageNet. This is why Strategy B (2.9~M trainable params) matches Strategy C (5.3~M trainable params) in final test accuracy.


\subsection{UMAP Feature Space}

UMAP~\cite{mcinnes2018umap} reduces the 1,280-dimensional penultimate-layer features to 2D while preserving local neighbourhood structure. Figure~\ref{fig:effnet_umap} plots the 500 test images in this reduced space.

The UMAP shows \textbf{compact, non-overlapping islands per species}. Biologically related groups (large swallowtails, blue Lycaenidae) appear as neighbouring macro-clusters, indicating the model has learned a latent space that mirrors real phylogenetic relationships. Contrast this with the CLIP embedding, which showed amorphous, heavily overlapping blobs — directly explaining the 30.8\% vs 97.0\% accuracy gap.

\begin{figure}[H]
    \centering
    \includegraphics[width=\linewidth]{effnet_umap_features.png}
    \caption{UMAP of fine-tuned EfficientNet features (500 test images, 100 species). Species form tight, well-separated clusters; biologically similar species (e.g., co-mimics) neighbour each other, reflecting the learned taxonomy.}
    \label{fig:effnet_umap}
\end{figure}

\section{Confusion Analysis}

The final test metrics for Strategy B are:
\begin{itemize}
    \item \textbf{Top-1 Accuracy:} 97.0\%
    \item \textbf{Top-3 Accuracy:} 99.2\%
    \item \textbf{Macro Precision:} 97.36\%
    \item \textbf{Macro Recall:} 97.0\%
    \item \textbf{Macro F1-score:} 96.92\%
\end{itemize}

\begin{figure}[H]
    \centering
    \includegraphics[width=\linewidth]{effnet_confusion_matrix.png}
    \caption{Confusion matrix restricted to the 20 most error-prone species. Confusions cluster around known mimicry pairs (e.g., Monarch/Viceroy, Adonis/Chalk Hill Blue), not across unrelated taxa.}
    \label{fig:conf_matrix}
\end{figure}

The confusion matrix (Figure~\ref{fig:conf_matrix}) restricts to the 20 highest-error species. All confusions occur between species with documented mimicry relationships or extreme morphological similarity. There are no cross-family errors (e.g., a Swallowtail mistaken for a Lycaenid), confirming that residual errors are irreducible without higher-resolution imaging.

\section{Deployment}

The trained weights for all three strategies are served from a public Hugging Face Hub repository (\texttt{Anurag29104/butterfly-classifier}). The Streamlit application downloads the selected model on first request and caches it with \texttt{@st.cache\_resource}. The sidebar exposes a radio selector to switch between Strategy A, B, and C in real time, with live Top-1, Top-3, and Macro-F1 metrics. All Wikipedia summaries, IUCN conservation statuses, and species fun facts are pre-cached locally to eliminate runtime HTTP latency.

\section{Conclusion}

This project demonstrates that for fine-grained scientific classification, zero-shot foundation models are insufficient: CLIP's 30.8\% accuracy reflects its lack of species-level visual vocabulary. By contrast, unlocking only the final convolutional block of EfficientNet-B0 — just 2.9~M of 4.14~M total parameters — achieves \textbf{97.0\% Top-1 accuracy}, matching a fully fine-tuned model at lower cost. Grad-CAM confirms that the model attends to biologically meaningful wing features, and the UMAP embedding reveals a taxonomically coherent latent space. The complete system is deployed as a real-time Streamlit application.
